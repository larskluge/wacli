# v0.2.1 — whatsmeow refresh

## Problem

`wacli sync` fails with WhatsApp server error 405 "Client outdated". The embedded client protocol version (`2.3000.1030403648`) has been deprecated by WhatsApp. Root cause: `go.mau.fi/whatsmeow` pinned to a Feb 2026 pseudo-version; a refresh is required. Tracked in steipete/wacli#106.

## Scope

- Upgrade **only** `go.mau.fi/whatsmeow` to latest. Do not touch other direct/indirect deps beyond what `go mod tidy` reshuffles as a consequence.
- Version bump: current in-source `0.5.0` → `0.2.1`. (The unreleased 0.5.0 bump is being rolled back; the last real release is 0.2.0, and 0.2.1 is the natural patch.)
- Update `CHANGELOG.md` and the README "Recent updates" section.

## Approach (TDD)

1. **Failing tests first** in `cmd/wacli`:
   - `version_test.go`: assert `version == "0.2.1"`.
   - `gomod_test.go`: parse `go.mod` and assert `go.mau.fi/whatsmeow` pseudo-version timestamp is `>= 20260327000000` (newer than the broken Feb 11 pin).
2. Run tests → both should fail.
3. `go get -u go.mau.fi/whatsmeow@latest` (must run **before** `go mod tidy`, per issue #106 comment).
4. `go mod tidy`.
5. Bump `version` constant in `cmd/wacli/root.go`.
6. Update `CHANGELOG.md` (new `0.2.1` entry) and README "Recent updates" heading.
7. Re-run `go test ./...` → green.
8. Build via `pnpm build` (uses `-tags sqlite_fts5` + the `CGO_CFLAGS` workaround).
9. **Manual runtime verification**: `./dist/wacli sync --once --idle-exit 5s` — confirm no 405 error.

## Non-goals

- No code changes beyond version string + tests.
- No Homebrew tap update (separate concern).
- No changes to other dependencies.
